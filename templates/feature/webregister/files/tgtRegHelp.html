Registering Your Target

In order to use your board you need to register it with your user account. Once registered, the board will automatically contact the Helix App Cloud servers whenever it is booted. When you login, you will see the board as "online".

There are 3 methods that can be used to register your board:

  • Create a Device using your login credentials and supplying a device name. This is the simplest in that it involves the fewest steps.
  • Have Helix App Cloud issue a PIN number for your board which you can the use to create a device using Helix App Cloud. This method requires additional steps; however, it can be done without login credentials.
  • Associate your board with a device that has already been created. This method is best when exchanging boards between users.

The sections below will describe each method.

Creating an Account

If you do not already have a Helix App Cloud account you should create one now by going to app.cloud.windriver.com or another URL you were supplied with. Select the Register Now button and follow the instructions provided.

Creating a Device

This is a one step process to add a device to your Helix App Cloud workspace. Select Register-->Create Device. You will be presented with a Dialog box here you should enter the following information.

Field Value Required
Username Enter your Helix App Cloud user name, such as joe@coolcompany.com Yes
Password Enter your Helix App Cloud password. This is never saved and always transmitted over a secure link. Yes
Device Name Give your board a name. This will become the Device name in your Helix App Cloud workspace Yes
SOCKS5 Proxy If a Proxy is required to communicate with cloud services you can use a SOCKS5 proxy. Enter a hostname and port using "hostname:port" No

After pressing the Create Device button, the board will exchange information with the Helix App Cloud servers. When complete you will see the named Device in your Helix App Cloud workspace and it will show Online if communication has been established.

Generating a PIN Code

This is a multi step process to add a device to your Helix App Cloud workspace. Start by selecting Register-->PIN Code. You will be presented with a Dialog box here you should enter the following information.

Field Value Required
Device Name Give your board a name. This will become the Device name in your Helix App Cloud workspace Yes
SOCKS5 Proxy If a Proxy is required to communicate with cloud services you can use a SOCKS5 proxy. Enter a hostname and port using "hostname:port" No

Upon pressing Generate PIN you will be presented with a highlighted PIN code and a countdown bar. The code must be use within the time shown by the countdown bar, typically 20 minutes.

In another browser window or tab, go to Helix App Cloud and select Create Device. Next select "Create a device using it's unique ID". Copy the highlighted PIN code and paste it into the dialog box on Helix App Cloud. Your device will appear in your workspace and should be Online.

Dismiss the "PIN Code" dialog using the Device Created button.

Should you decide not to use the PIN Code, your can select Cancel in the "PIN Code" Dialog box. Your board will remain unchanged.

Associate a Board with a Device ID

In some cases a Device already exists in your, or another persons, Helix App Cloud Workspace. It is possible to associate a physical board with that Device.

Choose Register-->Associate Board and you will be presented with a dialog box like the one below. Fill in the fields as follows:

Field Value Required
Device ID Copy the Device ID from a Helix App Cloud workspace that you wish to associate the physical board with into this field. Device IDs are a string that looks like "WR-TEST-F76D_bec719bb-d880-41a0-b2aa-e1a2ef1afb57". Yes
Server Address If you are using a non-standard Helix App Cloud server, enter the address of the server here. Most users will not use this No
SOCKS5 Proxy If a Proxy is required to communicate with cloud services you can use a SOCKS5 proxy. Enter a hostname and port using "hostname:port" No

Click the Associate button and you will see the Device in the Helix App Cloud Workspace come online.

Advanced Topics

Toggle Advanced Mode

If you wish to see additional information regarding board registration you can select Utilities-->Toggle Advanced.

HTTPS Proxy

Some sites require a proxy in order to communicate with HTTP(S) servers. When registering a board using any of the methods above, the board will attempt to communicate with Helix App Cloud using "https://" style URLs. To set the correct path to the HTTP(S) Proxy at your facility, select Utilities-->Manage Proxy. The URL you enter is probably the same HTTPS proxy URL that is set in your web browser.

Note that the HTTPS proxy is not the same as the SOCKS5 proxy. The SOCKS5 proxy is used to establish a persistent link between your board and Helix App Cloud to facilitate debugging and analysis tools. HTTPS proxy only supports a transaction-style link and is used only during the board registration process.
