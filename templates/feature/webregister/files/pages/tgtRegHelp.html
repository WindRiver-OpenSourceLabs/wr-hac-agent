Registering Your Target

When you create a device in Helix App Cloud, you can associate that device with either simulated hardware or a physical target. Simulated hardware is automatically registered and available for use with your device. However, physical targets must be registered with your user account. Once registered, a target board automatically contacts the Helix App Cloud server any time it is booted. When you log into Helix App Cloud, the target displays its status as "online".

Register your board using one of the following methods:

  • Method 1: Use your login credentials to create a Helix App Cloud device and register your target board. (This is the simplest method.)
  • Method 2: Request that Helix App Cloud issue a PIN number for your board. Use this PIN to create a Helix App Cloud device and register the target. While this method requires additional steps, it allows you to register the board without login credentials.
  • Method 3: Associate your board with an existing Helix App Cloud device. This method works well when exchanging boards between users.

For information on reviewing your target settings, accessing advanced settings, and setting up an HTTPS proxy, see More Information.

Before You Begin

If you have not already done so, create a Helix App Cloud account by going to app.cloud.windriver.com or another URL, if you are supplied with one. Click Register Now and follow the instructions provided.

Method 1: Register the Target Using Your Login Credentials

This method adds a device to your Helix App Cloud workspace and registers your target.

  1. On the target-based web page, select Register --> Create Device.

    2. The following dialog appears:

  2. Enter the required information.
    3. Field Value Required
    User Name Enter your Helix App Cloud username. For example, joe@coolcompany.com. Yes
    Password Enter your Helix App Cloud password. Your password is never saved and is always transmitted over a secure link. Yes
    Device Name Provide a descriptive name for your target board. This name is displayed as your Helix App Cloud device name in the Helix App Cloud workspace. Yes
    Target Agent SOCKS5 Proxy If you require a proxy to communicate with cloud services, use a SOCKS5 proxy. Enter a host name and port using the following format: "hostname:port". No
  3. Click Create Device.

    4. The board exchanges information with the Helix App Cloud server. When complete, the named device appears in your Helix App Cloud workspace. If communication is successfully established, the status is shown as Online in the Helix App Cloud workspace.

Method 2: Register the Target Using a Generated PIN Code

This process adds a device to your Helix App Cloud workspace and registers your target without requiring your Helix App Cloud credentials.

  1. On the target-based web page, select Register --> Generate PIN.

    2. The following dialog box appears:

  2. Enter the required information.
    3. Field Value Required
    Device Name Provide a descriptive name for your target board. This name is displayed as your Helix App Cloud device name in the Helix App Cloud workspace. Yes
    Target Agent SOCKS5 Proxy If you require a proxy to communicate with cloud services, use a SOCKS5 proxy. Enter a host name and port using the following format: "hostname:port". No
  3. Click Get PIN.

    4. You are presented with a highlighted PIN code and a countdown bar. Note that the code must be use within the time shown by the countdown bar; typically, 20 minutes.

Using the PIN

To use the pin, in another browser window or tab, go to Helix App Cloud and perform the following steps.

  1. Click Create Device.
  2. Select "Create a device using it's unique ID".
  3. Copy the highlighted PIN code and paste it into the dialog box on Helix App Cloud.

    4. The device appears in your workspace and shows a status of Online.

  4. Click Device Created.

    5. This acknowledges that you have created a device on Helix App Cloud.

    NOTE: If you do not wish to use the PIN Code after it is generated, click Cancel in the "PIN Code" dialog box. The target is not registered and your board remains unchanged.

Register the Target by Associating it with an Existing Device

If you, or another person, already have an existing Helix App Cloud device prepared for the target in your Helix App Cloud workspace, you can associate that device with your physical target to register the board.

  1. On the target-based web page, select Register --> Associate Board.

    2. The following dialog box appears:

  2. Enter the required information.
    3. Field Value Required
    Device ID Enter the Device ID for the Helix App Cloud device that you want to associate with the physical board. The device ID is a string similar to the following: "WR-TEST-F76D_bec719bb-d880-41a0-b2aa-e1a2ef1afb57". Yes
    Server Address If you are using a non-standard Helix App Cloud server, enter the address of the server here. (This is not applicable to most users.) No
    SOCKS5 Proxy If you require a proxy to communicate with cloud services, use a SOCKS5 proxy. Enter a host name and port using the following format: "hostname:port". No
  3. Click Associate.

    4. The device comes online in the Helix App Cloud workspace.

    More Information

    Device Status Page

    The target-based website always shows the current board settings. You can refresh the settings at any time by clicking the Refresh icon in the upper left corner of the menu bar. The page typically refreshes whenever a change is made to the settings.

    Toggle Advanced Mode

    To see additional information regarding board registration, select Utilities --> Toggle Advanced.

    HTTPS Proxy

    Some sites require a proxy in order to communicate with an HTTP(S) server. When registering a board using any of the methods provided, the target attempts to communicate with Helix App Cloud using "https://" style URLs. To set the correct path to the HTTP(S) proxy at your facility, select Utilities --> Manage Proxy. Typically, the URL you need to enter is the same HTTPS proxy URL that is set in your web browser.

    NOTE: The HTTPS proxy is not the same as the SOCKS5 proxy. The SOCKS5 proxy is used to establish a persistent link between your board and Helix App Cloud to facilitate debugging and analysis tools. The HTTPS proxy only supports a transaction-style link and is used only during the board registration process.
